# AGENTS.md

## Commands

```bash
go build -o linutils-rakesh main.go   # produces single binary
go run main.go                          # run without building
```

No tests, no CI, no lint/format configs exist.

## Architecture

- `main.go` — orchestrator: detects system, runs TUI, dispatches selected features via switch
- `internal/system/detect.go` — `GetSystemInfo()` returns `Info{OS, OSVersion, DE, DEVersion, SessionType}` from `/etc/os-release` + env vars
- `internal/pkgmanager/manager.go` — `PackageManager` interface (`Install`, `Remove`, `Update`, `Upgrade`, `IsInstalled`), implemented by `AptManager`, `PacmanManager`, `DnfManager`
- `internal/tui/form.go` — `RunMainMenu()` + feature string constants used as dispatch keys
- `internal/tui/debloat_ui.go` — standalone Bubble Tea model for the debloat checklist (uses `tea.Quit`, not `huh`)
- `internal/modules/` — feature implementations (software, git, github, shell, gnome apps/services/extensions, debloat)
- `internal/config/` — DE/WM configurators (`HyprlandConfigurator`, `I3Configurator`, `GnomeConfigurator`) implementing `EnvConfigurator` interface
- `data/` — empty directory (reserved for future use)

## Adding a new feature

1. Implement logic in `internal/modules/` or `internal/config/`
2. Add a `FeatureXxx` constant in `internal/tui/form.go`
3. Add `huh.NewOption(...)` in the `RunMainMenu` form
4. Add a `case tui.FeatureXxx:` in `main.go`'s switch

## Distro quirks (hard-earned)

- **bat**: package is `batcat` on Debian/Ubuntu, `bat` elsewhere (`main.go:70-74`)
- **GitHub CLI**: package is `gh` on apt/dnf, `github-cli` on pacman (`github.go:16-23`)
- **Debloat Extensions**: only offered on Fedora + Gnome (`debloat.go:21-23`)
- **User services**: `tracker-miner-fs-3`, `tracker-extract-3`, `gnome-software` are masked with `systemctl --global` (user-scoped); others use system scope (`gnome_services.go:53-64`)
- **DNF check-update**: returns exit code 100 when updates exist, which is intentionally ignored (`dnf.go:42`)
